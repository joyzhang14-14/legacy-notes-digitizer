# 教案数字化工具 v2 — 新架构

## 架构设计原则

1. **先识别结构，后处理内容** — 在原始图上做结构扫描，不做任何预处理
2. **插图和文字走独立通道** — 插图保留原始对比度，文字单独增强
3. **从白纸创建全新文档** — 不在扫描件上叠加，而是完全重建
4. **多轮 AI 校验** — Opus 模型多遍扫描，准确度优先
5. **跨平台字体** — 使用开源思源字体，pip 安装，不依赖系统

## 执行流程

```
Phase 0 → Phase 1 → Phase 2a (并行) → Phase 3
                   → Phase 2b (并行) ↗
```

| Phase | 脚本 | AI模型 | 功能 |
|-------|------|--------|------|
| Phase 0 | orientation.py | Opus | 检测旋转方向 → 自动纠正 |
| Phase 1 | structure.py | Opus ×2轮 | 在原始图上扫描结构 → 输出位置编码JSON |
| Phase 2a | illustration.py | 无（纯CV） | 插图通道：原始对比度提取 → trace增强线稿 |
| Phase 2b | text_ocr.py | Opus ×3轮 | 文字通道：区域增强 → OCR → 多轮校验 |
| Phase 3 | build.py | 无 | 从白纸合成：PDF(图层) + PPT + 纯文字 |
